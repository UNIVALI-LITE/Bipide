BIP IV

Architecture

The architecture of the BIP IV considers the architecture of its predecessors, adding further support for procedures, the possibility of performing manipulation of vectors and Control input and output. To support such functionality was necessary to add four more registers, which they are, (i) the Indr registrar, responsible for storing the base indict the support vectors, (ii) the SP register, which has the function of indicating the stack when calling a procedure, (iii) the IN_PORT registers and (iv) OUT_PORT responsible for mapping input and output, respectively. (RECH, 2011).
